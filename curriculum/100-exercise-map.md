# 100 TypeScript PR Review Exercise Map

This is the curriculum-first blueprint. Each row is one separate PR review exercise. The full exercise object for each case must later be authored using [the exercise standard](../docs/exercise-authoring-standard.md), including changed files, exact line references, three hints per flaw, and the final expert debrief.

Progression principle:

- TS-001 to TS-020: learn to read contracts, state, and migrations.
- TS-021 to TS-040: learn auth, permissions, and API/service boundary review.
- TS-041 to TS-070: learn async systems, event ingestion, idempotency, and failure modes.
- TS-071 to TS-090: learn scaling, ownership boundaries, and architecture erosion.
- TS-091 to TS-100: principal-level synthesis across product, data, contracts, rollout, and operations.
