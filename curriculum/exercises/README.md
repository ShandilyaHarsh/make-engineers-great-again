## Learner-Ready Definition

An exercise is learner-ready only when it includes:

- metadata and target diff size,
- source repo and real codebase grounding,
- PR description,
- changed files and review surface,
- realistic degraded diff of at least 500 represented lines,
- 2-3 intended flaws,
- exactly three hints per flaw,
- expected identify/impact/fix answer per flaw,
- final expert debrief covering product-level change, changed contracts, failure modes, reviewer thought process, and better implementation direction,
- correctness verdict rubric.
